# Use Cases

## 1. Multi-device Obsidian + OpenClaw workflow

You use OpenClaw on more than one macOS device and want a cleaner way to keep Obsidian-related configuration in sync through iCloud.

Relevant module:

- `skills/obsidian-openclaw-sync`

## 2. Quick operational visibility

You want a lightweight way to check recent failures and rough API cost signals without setting up a full monitoring stack.

Relevant module:

- `skills/openclaw-skill-observability`

## 3. Cost awareness for daily agent usage

You need a minimal way to audit token usage and estimate spend so that OpenClaw stays sustainable in regular use.

Relevant module:

- `skills/openclaw-cost-auditor`
