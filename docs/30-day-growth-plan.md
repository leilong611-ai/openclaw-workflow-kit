# 30-Day Growth Plan

## Goal

Build the first layer of visibility and trust for `openclaw-workflow-kit`.

## Week 1

- Publish the repository baseline
- Publish the first release
- Add issue templates
- Add contributing guidance
- Post first launch announcement

## Week 2

- Add screenshots or GIFs
- Publish one Chinese long-form introduction
- Publish one short Discord / community intro
- Open one discussion asking for the next practical workflow to package

## Week 3

- Add one new use case
- Respond to all early feedback
- Refine README based on first questions
- Post an update with one concrete before/after workflow story

## Week 4

- Publish `v0.1.1` or `v0.2.0`
- Add one more practical module or documentation bundle
- Summarize user feedback into roadmap updates
- Prepare ecosystem publishing materials

## What drives stars

- Clear first impression
- Real workflow value
- Visible maintenance
- External distribution
- Fast response to feedback

## What to avoid

- Empty hype
- Overpromising
- Shipping too many unrelated modules at once
- Turning the repo into a random collection
